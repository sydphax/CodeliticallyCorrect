Codelitically Correct

Accessibility

  • Use the alt attribute in image tags so screen readers can get a verbal description
  • Apply responsive web design techniques to make the site flow on various device screens
  • Add a favicon (small icon) to the head element to help visitors identify the page when multiple tabs are open
  • Have a title in the head element to display on the tab and aid in search engine optimization

Syntax

  • Employ specific tags such as <article> or <address>, rather than <div> to make code more organized and easier to read
  • Specify the doctype to allow browsers to render properly and markup validators to check the correct rules
  • Add the language attribute so search engines filter results better and screen readers get proper translations
  • Stick to lowercase letters for consistency (plus it looks better)
  • Use either single quotation marks or double quotation marks throughout the whole thing

Formatting

  • Put external style sheets in the head element to reduce render times
  • Add a description metatag with keywords so search engine crawlers pull more instances of the site
  • Let CSS handle any <br> elements
  • Keep indentation consistent (spaces or tabs) and have all tags line up correctly for readability

Bonus Tips

  • Validate code frequently to ensure the website is free of errors without going unnoticed
  • Check the browser developer tools to see how the website looks on many different viewports
  • Leave comments to understand the purpose behind code, to section things, and to make it more readable in the future
